# Ride Demand prediction Data-analysis(Interaction Dashboard creation using Power BI)

## Project Objective

The ride apps like uber,ola,rapido,etc wants to create an report on total rides and demand areas.so, they can know the  demand areas and the total no.of sufficient / insufficient of the drivers. 
## Dataset used
- <a href="https://github.com/kavyachallagundla54-crypto/Data-analysis-dashboard/blob/main/fairfare_ride_demand_dataset.csv">Dataset</a>

## Questions(KPIs)
- Compare the demand level with cities using barchart.
- which city has highest demand?
- compare the availability of drivers with cities using piechart.
- compare the fare accepentance with ride distance.
- compare the final fare with demand in weekend or weekdays.
- compare the traffic delay with day of week.


  -Dashboard Interaction <a href ="https://github.com/kavyachallagundla54-crypto/Data-analysis-dashboard/blob/main/Screenshot%202026-03-15%20112444.png"> Dashboard </a>

## Process
- Verify data for any missing values and annomalies and sort out the same.
- Made sure data is consistent and clean  with respect to datatype,dataformat and values used.
- created the pivot tables and charties according to the questions asked.
- created interactive dashboard using the data.
## Project Insights
- Peak Ride demand time
- High Fare during peak hours
- weekly patterns
- Traffic delay affects ride efficiency
